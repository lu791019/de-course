## 第一部分：啟動 MySQL + phpMyAdmin

### Step 1：建立共用網路 my_network

`mysql.yml`、`rabbitmq-network.yml`、Worker、Producer **全部都掛在 `my_network` 這張外部網路**，
彼此才能用「服務名稱」互相找到（例如 Worker 用 `mysql` 找到資料庫）。

```bash
docker network create my_network
```

✅ **預期**：印出一串網路 ID（如果出現 `already exists` 代表先前建過，忽略即可）

```bash
# 確認網路存在
docker network ls | grep my_network
```

### Step 2：啟動 MySQL + phpMyAdmin

```bash
cd ~/stock-crawler
docker compose -f mysql.yml up -d
```

✅ **預期**：拉取 `mysql:8.0` 與 `phpmyadmin:5.2` 映像後，兩個容器都啟動。

```bash
docker compose -f mysql.yml ps
```

看到 `mysql` 和 `phpmyadmin` 兩個服務都是 `Up` 狀態。

### Step 3：看 MySQL log 確認初始化完成

MySQL 第一次啟動要初始化資料目錄，需等十幾秒：

```bash
docker compose -f mysql.yml logs mysql | tail -20
```

✅ **預期**：看到 `ready for connections` 才代表資料庫可以連了。

> ⚠️ 常見坑：剛 `up` 馬上連線會失敗（`Connection refused`），那是 MySQL 還在初始化。
> 看到 `ready for connections` 再進行下一步。

### mysql.yml 的關鍵設定

| 設定 | 值 | 說明 |
|------|-----|------|
| `MYSQL_DATABASE` | `mydb` | 啟動時自動建立的資料庫 |
| `MYSQL_USER` / `MYSQL_PASSWORD` | `user` / `ppWgnb_mfGe2m_` | 一般使用者 |
| `MYSQL_ROOT_PASSWORD` | `ppWgnb_mfGe2m_` | root 密碼 |
| ports | `3306:3306` | 對外開放 MySQL 連線埠 |
| phpMyAdmin ports | `8080:80` | Web 介面在 `localhost:8080` |
| `PMA_HOST` | `mysql` | phpMyAdmin 用服務名稱連 MySQL |

## 第三部分：建立資料庫和資料表

> 💡 重要觀念：本專案的爬蟲程式用 `pandas.to_sql()` 寫入時會**自動建表**，
> 所以實務上你「不一定要手動建表」。但這一部分先帶你手動建一次，
> 理解資料庫（database）和資料表（table）的關係。

### 方法 A：用 phpMyAdmin 下 SQL（推薦先練這個）

在 phpMyAdmin 上方點 **SQL** 頁籤，貼上以下指令並執行：

```sql
-- 1. 建立資料庫（mydb 已存在，這行示範語法用，可跳過）
CREATE DATABASE IF NOT EXISTS mydb;

-- 2. 切換到 mydb
USE mydb;

-- 3. 建立台股股價資料表
CREATE TABLE IF NOT EXISTS TaiwanStockPrice (
    date              DATE,
    stock_id          VARCHAR(10),
    Trading_Volume    BIGINT,
    Trading_money     BIGINT,
    open              FLOAT,
    max               FLOAT,
    min               FLOAT,
    close             FLOAT,
    spread            FLOAT,
    Trading_turnover  BIGINT
);
```

✅ **預期**：執行成功，左側 `mydb` 底下出現 `TaiwanStockPrice` 資料表。

### 欄位對照（來自 FinMind TaiwanStockPrice）

| 欄位 | 型別 | 說明 |
|------|------|------|
| `date` | DATE | 交易日 |
| `stock_id` | VARCHAR | 股票代碼 |
| `Trading_Volume` | BIGINT | 成交量（股） |
| `Trading_money` | BIGINT | 成交金額 |
| `open` / `max` / `min` / `close` | FLOAT | 開 / 高 / 低 / 收 |
| `spread` | FLOAT | 漲跌 |
| `Trading_turnover` | BIGINT | 成交筆數 |

> 📌 如果你想「完全交給程式自動建表」，可以**跳過手動建表**，
> 直接進第五部分跑爬蟲，`to_sql()` 會依 DataFrame 欄位自動建出 `TaiwanStockPrice`。

## 第四部分：用 Python 連線 MySQL

爬蟲程式是用 **SQLAlchemy** 連 MySQL。先用一小段程式確認本地能連得上。

### Step 1：看連線設定來自哪裡

```bash
cat crawler/config.py
```

重點：MySQL 連線參數全部用 `os.environ.get(key, default)` 讀取。

| 變數 | 預設值 | 說明 |
|------|--------|------|
| `MYSQL_HOST` | `127.0.0.1` | 本地直連用 127.0.0.1；Docker 內改成 `mysql` |
| `MYSQL_PORT` | `3306` | 連線埠 |
| `MYSQL_ACCOUNT` | `root` | 帳號 |
| `MYSQL_PASSWORD` | `ppWgnb_mfGe2m_` | 密碼 |

### Step 2：測試連線

```bash
cd ~/stock-crawler
uv run python -c "
from sqlalchemy import create_engine, text
from crawler.config import MYSQL_ACCOUNT, MYSQL_PASSWORD, MYSQL_HOST, MYSQL_PORT

address = f'mysql+pymysql://{MYSQL_ACCOUNT}:{MYSQL_PASSWORD}@{MYSQL_HOST}:{MYSQL_PORT}/mydb'
engine = create_engine(address)

with engine.connect() as conn:
    result = conn.execute(text('SELECT VERSION()'))
    print('MySQL 版本:', result.scalar())
    print('連線成功！')
"
```

✅ **預期**：

```
MySQL 版本: 8.0.xx
連線成功！
```

> ⚠️ 連線字串格式：`mysql+pymysql://帳號:密碼@主機:埠/資料庫`
> `pymysql` 是 Python 連 MySQL 的驅動，已透過 `uv sync` 裝好。

### Step 3：看資料表（目前是空的）

```bash
uv run python -c "
from sqlalchemy import create_engine, text
from crawler.config import MYSQL_ACCOUNT, MYSQL_PASSWORD, MYSQL_HOST, MYSQL_PORT
engine = create_engine(f'mysql+pymysql://{MYSQL_ACCOUNT}:{MYSQL_PASSWORD}@{MYSQL_HOST}:{MYSQL_PORT}/mydb')
with engine.connect() as conn:
    rows = conn.execute(text('SHOW TABLES')).fetchall()
    print('資料表:', [r[0] for r in rows])
"
```

✅ **預期**：看到 `['TaiwanStockPrice']`（若第三部分有建表）或 `[]`（還沒建）。

## 第五部分：爬蟲資料寫入 MySQL

這是本集核心。完整流程：**啟動 infra → 啟動 Worker → 發 Producer → 看 Flower → 查 DB**。

### 寫入是怎麼發生的？

看 `crawler/tasks_crawler_finmind.py` 的 `crawler_finmind` task：

```python
@app.task()
def crawler_finmind(stock_id):
    resp = requests.get(url, params=parameter)   # 1. 爬 FinMind
    df = pd.DataFrame(data["data"])              # 2. 轉成 DataFrame
    upload_data_to_mysql(df)                     # 3. 寫入 MySQL
```

而 `upload_data_to_mysql()` 就是用 SQLAlchemy + `df.to_sql(..., if_exists="append")`，
把整個 DataFrame 一次寫進 `TaiwanStockPrice` 表。

### Step 1：啟動基礎服務（RabbitMQ + MySQL）

MySQL 第一部分已啟動，這裡再補上 RabbitMQ：

```bash
cd ~/stock-crawler
docker compose -f rabbitmq-network.yml up -d
docker compose -f mysql.yml up -d        # 若已啟動會顯示 up-to-date
```

```bash
# 確認都在同一張網路上
docker compose -f rabbitmq-network.yml ps   # rabbitmq + flower
docker compose -f mysql.yml ps              # mysql + phpmyadmin
```

✅ **驗證 Web 介面**：

```bash
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:15672   # RabbitMQ → 200
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:5555    # Flower   → 200
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:8080    # phpMyAdmin → 200
```

### Step 2：啟動 Worker（會連 MySQL）

```bash
docker compose -f docker-compose-worker-network.yml up -d
```

✅ **預期**：兩個 Worker（`crawler_twse`、`crawler_tpex`）啟動。

```bash
docker compose -f docker-compose-worker-network.yml logs | tail -30
```

看到 Worker `ready`，並印出環境變數含 `MYSQL_HOST: mysql`（不是 127.0.0.1）。
這個 compose 在 environment 裡設了 `MYSQL_HOST=mysql`、`RABBITMQ_HOST=rabbitmq`，
讓容器用**服務名稱**連到 infra。

### Step 3：發任務（Producer 寫入 DB 版本）

```bash
docker compose -f docker-compose-producer-network.yml up
```

✅ **預期**：Producer 印出 `send task_2330 task` / `send task_00679b task` 後結束（exit 0）。

這支 Producer 跑的是 `producer_multi_queue.py`，內容：

```python
from crawler.tasks_crawler_finmind import crawler_finmind   # 寫 DB 版的 task

task_2330 = crawler_finmind.s(stock_id="2330")
task_2330.apply_async(queue="twse")        # 上市 → twse worker
task_00679b = crawler_finmind.s(stock_id="00679B")
task_00679b.apply_async(queue="tpex")      # 上櫃/美債 → tpex worker
```

> 一次性服務跑完 `Exited(0)` 是正常的，Producer 的工作就是「發完任務就走」。

### Step 4：確認 Worker 真的執行成功

```bash
docker compose -f docker-compose-worker-network.yml logs | tail -40
```

✅ **預期**：看到 `Task crawler.tasks_crawler_finmind.crawler_finmind[...] succeeded`，
而不是只有 `received` 或 `FAILURE`。

### Step 5：用 Flower 確認任務狀態

瀏覽器打開 `http://localhost:5555` → **Tasks** 頁籤。

✅ **預期**：看到 2 筆 `crawler_finmind`，狀態都是 **SUCCESS**（不是 FAILURE / PENDING）。

## 第六部分：用 SQL 查詢資料

任務成功後，資料應該已經進 `TaiwanStockPrice` 表了。

### 方法 A：phpMyAdmin 查詢

打開 `http://localhost:8080` → 左側點 `mydb` → `TaiwanStockPrice` → **SQL** 頁籤：

```sql
-- 看資料表有幾筆
SELECT COUNT(*) FROM TaiwanStockPrice;

-- 看台積電最近 10 筆
SELECT date, stock_id, open, close, Trading_Volume
FROM TaiwanStockPrice
WHERE stock_id = '2330'
ORDER BY date DESC
LIMIT 10;
```

✅ **預期**：COUNT 大於 0，並看到 2330 的股價資料。

### 方法 B：Python 查詢

```bash
cd ~/stock-crawler
uv run python -c "
import pandas as pd
from sqlalchemy import create_engine
from crawler.config import MYSQL_ACCOUNT, MYSQL_PASSWORD, MYSQL_HOST, MYSQL_PORT

engine = create_engine(f'mysql+pymysql://{MYSQL_ACCOUNT}:{MYSQL_PASSWORD}@{MYSQL_HOST}:{MYSQL_PORT}/mydb')
df = pd.read_sql('SELECT * FROM TaiwanStockPrice', con=engine)
print(f'總筆數: {len(df)}')
print(df.groupby('stock_id').size())          # 各股票各幾筆
print(df.head())
"
```

✅ **預期**：印出總筆數、各 `stock_id`（2330 / 00679B）的筆數，以及前幾筆資料。

> 📌 端到端閉環走完了：**爬蟲 → 任務佇列 → Worker → MySQL → 查得到資料**。
> 這就是資料工程最基本的 ingestion pipeline。

## 第七部分：環境切換（本地 vs Docker）

同一份 `config.py`，怎麼做到「本地直接跑」和「Docker 容器內跑」都能連對 MySQL？
答案就是 `os.environ.get(key, default)` 這個模式。

### 核心原理

```python
# crawler/config.py
MYSQL_HOST = os.environ.get("MYSQL_HOST", "127.0.0.1")
```

| 情境 | 有無設環境變數 | 實際用的值 |
|------|--------------|-----------|
| **本地直跑**（Part 4 / Part 6） | 沒設 | 預設 `127.0.0.1` → 連 host 上的 MySQL |
| **Docker 容器內**（Worker） | compose 設了 `MYSQL_HOST=mysql` | `mysql` → 連同網路的 mysql 服務 |

### 為什麼容器內不能用 127.0.0.1？

容器內的 `127.0.0.1` 指的是**容器自己**，不是你的電腦。
所以容器要連 MySQL，得用 Docker 網路裡的**服務名稱** `mysql`，
靠的就是大家都在 `my_network` 這張網路上。

### 看 Worker compose 怎麼覆蓋

```yaml
# docker-compose-worker-network.yml
services:
  crawler_twse:
    environment:
      - RABBITMQ_HOST=rabbitmq    # 覆蓋預設 127.0.0.1
      - MYSQL_HOST=mysql          # 覆蓋預設 127.0.0.1
    networks:
      - my_network
```

### 一句話總結

> **預設值給本地開發用，環境變數給部署環境覆蓋。**
> 程式碼一行都不用改，換環境只換環境變數。這就是 12-Factor App 的 Config 原則。

### 收尾：關閉服務

```bash
cd ~/stock-crawler
docker compose -f docker-compose-worker-network.yml down
docker compose -f rabbitmq-network.yml down
docker compose -f mysql.yml down       # 注意：data volume 會保留，資料不會消失
```

> 💡 `down` 不會刪 volume，下次 `up` 資料還在。
> 要連資料一起清掉才加 `down -v`（謹慎使用）。
